## 开发规范

### Git提交规范
```
feat: 添加新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
test: 测试相关
chore: 构建/工具相关
```

### 代码审查
- 所有PR需要Review
- 关键模块需要双人Review
- 测试覆盖率 > 70%

### 发布流程
1. 完成开发
2. 代码审查
3. 测试验证
4. 打Tag发布
5. 更新文档
